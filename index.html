Slide Machine

Introduction

The intent of "slide machine" is to provide a simple and smart way to generate slides in various formats and layouts. A slide is just a two dimensional container of customizable components to meet specific requirements. The minimum viable product comprises just a background which is the core element of a slide. This project examines the power of vector graphics in web browsers and is therefore focused on SVG.

Requirements

  • 1. A background is based on responsive design rules which means that the content of the background scales to the window respectively.

  • 1.1. A simple index page with a link to a sample svg file which points back to the index page.
  • 1.1.1. Example: 1.1.1.svg

  • 2. A background can be customized by changing the color.

  • 2.1. See base color picker with square panels (4 only for now) in html.
  • 2.1.1. Example: 2.1.1.html
  • 2.2. Click on square with desired color to get a preview in full screen
  • 2.2.1. Example: 2.2.1.html

  • 3. The background color can be changed to radial gradient with two colors.

  • 3.1. A simple svg page with radial gradient and two colors.
  • 3.1.1. Example: 3.1.1.svg
  • preserveAspectRatio="none"

  • 4. The background color can be changed to linear gradient with two colors.

  • 4.1. A simple svg page with linear gradient and two colors.
  • 4.1.1. Example: 4.1.1.svg
  • preserveAspectRatio="none" patternUnits="userSpaceOnUse"

  • 5. The background is filled with a recangular pattern and one single color.

  • 5.1. A simple svg page with a tiled pattern.
  • 5.1.1. Example: 5.1.1.svg
  • The responsive behavior of the background is controlled by these attributes:

    In "svg": preserveAspectRatio="none"
    In "pattern": patternUnits="userSpaceOnUse"

    The vertical and horizontal tiling of the background is defined by changing these attributes:

    In "svg": viewBox="0 0 5 3"

    The attributes "width" and "height" of the "rect" element which contains the background need to be changed accordingly:

    In "rect": width="5" height="3"

    The gap between tiles of the pattern is defined by y and y in "rect" respectively:

    In "rect" nested in "pattern": y="0.05" x="0.05"
    In "rect" which contains the background: y="-0.05" x="-0.05"
